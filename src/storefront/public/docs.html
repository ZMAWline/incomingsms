API documentation

Built for automation and AI agents — every account has an API token. Rent a real US number with one call, poll its inbox for the OTP with another. No browser, no scraping.

Authentication

All authenticated endpoints accept your account's API token as a bearer token:

Authorization: Bearer $TOKEN

Find the token in the app under API access (it is also returned by GET /api/me). The examples below assume two shell variables:

BASE=https://your-numbox-host   # the storefront origin
TOKEN=0123abcd...               # your API token

If a token leaks, rotate it with POST /api/token/rotate — the old token stops working immediately. The browser app's session cookie works on the same endpoints, so everything below can also be called from logged-in frontend code.

Endpoints

GET/api/stockpublic

List rentable numbers. Each item carries the per-duration price in cents under prices (day / week / month). Numbers are masked until rented.

curl -s $BASE/api/stock
{ "stock": [ { "sim_id": 42, "carrier": "AT&T", "area_code": "347",
    "masked_number": "(347) •••-••43",
    "prices": { "day": 300, "week": 1500, "month": 4500 } } ] }
POST/api/rentbearer / cookie

Rent a number. Body parameters:

  • sim_id (integer, required) — from /api/stock
  • duration (string, optional) — "day" (24 h, default), "week" (7 days), or "month" (30 days); anything else is a 400

The price for the chosen duration is debited from your balance. The response reveals the full number in rental.e164.

curl -s -X POST $BASE/api/rent \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"sim_id": 42, "duration": "week"}'
GET/api/rentalsbearer / cookie

Your rentals, newest first, each with e164, duration, ends_at, and effective_status (active or ended).

curl -s -H "Authorization: Bearer $TOKEN" $BASE/api/rentals
GET/api/rentals/:id/messagesbearer / cookie

The rental's inbox, newest first. Poll this endpoint — OTP codes arrive in the message body within seconds of being sent. A few seconds between polls is plenty.

curl -s -H "Authorization: Bearer $TOKEN" $BASE/api/rentals/1234/messages
{ "messages": [ { "from_number": "+1855...", "body": "Your code is 482913",
    "received_at": "2026-06-13T18:02:11Z" } ] }
POST/api/depositsbearer / cookie

Top up your balance. Body: amount_cents (integer, minimum 1000 = $10). With crypto payments enabled the response includes an invoice_url to pay; the balance credits automatically on confirmation.

curl -s -X POST $BASE/api/deposits \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"amount_cents": 2500}'
GET/api/mebearer / cookie

Your account: email, balance_cents, and the current api_token. Handy as a cheap auth check before a run.

curl -s -H "Authorization: Bearer $TOKEN" $BASE/api/me
POST/api/token/rotatebearer / cookie

Replace your API token. The response carries the new token; the old one is dead the moment this call returns. Use it whenever a token may have leaked.

curl -s -X POST -H "Authorization: Bearer $TOKEN" $BASE/api/token/rotate
{ "ok": true, "api_token": "f3a9c0..." }

Example: an agent fetches an OTP

The whole loop — pick the cheapest number, rent it for a day, poll the inbox until a 4–8 digit code shows up:

# 1. cheapest available number
SIM_ID=$(curl -s $BASE/api/stock \
  | jq '.stock | sort_by(.prices.day) | .[0].sim_id')

# 2. rent it for a day
RENTAL=$(curl -s -X POST $BASE/api/rent \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d "{\"sim_id\": $SIM_ID, \"duration\": \"day\"}")
RENTAL_ID=$(echo "$RENTAL" | jq '.rental.id')
echo "number: $(echo "$RENTAL" | jq -r '.rental.e164')"

# 3. use the number wherever, then poll until a code lands
while :; do
  CODE=$(curl -s -H "Authorization: Bearer $TOKEN" \
    $BASE/api/rentals/$RENTAL_ID/messages \
    | jq -r '.messages[].body' | grep -oE '[0-9]{4,8}' | head -1)
  [ -n "$CODE" ] && break
  sleep 5
done
echo "OTP: $CODE"

Errors

Errors are JSON: { "error": "<code>" } with a matching HTTP status.

StatusErrorMeaning
401unauthorizedMissing or invalid token / session. Check the Authorization: Bearer header, or rotate the token if you lost it.
402insufficient_balanceThe rental price exceeds your balance — top up via POST /api/deposits and retry.
409just_takenSomeone rented that number between your stock check and rent call. Re-fetch /api/stock and pick another.
Rate limits may apply to keep the service fast for everyone. Abuse — including using numbers for fraud or spam — leads to account bans per our acceptable-use policy. Questions? support@otpdock.io.
